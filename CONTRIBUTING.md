Техническое руководство по архитектуре, расширению и модификации мессенджера для локальной сети.

## Общая архитектура

Проект построен по модульному принципу с чётким разделением ответственности:

- **`config.py`** — конфигурация и низкоуровневые утилиты;
- **`network.py`** — сетевая логика (UDP‑сокеты, потоки);
- **`message_handler.py`** — обработка бизнес‑логики сообщений;
- **`main.py`** — точка входа и пользовательский интерфейс.

## Модули и их интерфейсы

### 1. `config.py`

**Ответственность**:  
- Хранение констант.  
- Определение локального IP.

**Ключевые элементы**:  
- `PORT`: порт UDP (int).  
- `TIMEOUT`: таймаут сокета (float).  
- `BUFFER_SIZE`: размер буфера приёма (int).  
- `get_local_ip()`: функция возврата строкового IP.  
- `LOCAL_IP`: кэшированный результат `get_local_ip()`.

### 2. `network.py`

**Ответственность**:  
- Отправка UDP‑пакетов.  
- Приём сообщений в отдельном потоке.  
- Делегирование обработчику.

**Класс `NetworkManager`**  
**Конструктор**:  
```python
NetworkManager(on_message_received: Callable[[str, str], None])
```
- `on_message_received` — коллбэк: `def handler(sender_ip: str, message: str)`.

**Методы**:  
- `send_message(message: str, target_ip: str)` — отправляет сообщение.  
- `start_listening()` — запускает поток приёма.  
- `_listen()` — внутренний цикл приёма (работает в потоке).

### 3. `message_handler.py`

**Ответственность**:  
- Форматирование и вывод сообщений.  
- Будущая валидация/фильтрация.

**Функция `handle_message`**  
```python
handle_message(sender_ip: str, message: str)
```

### 4. `main.py`

**Ответственность**:  
- Инициализация компонентов.  
- Цикл ввода пользователя.  
- Обработка исключений.

**Точка входа `main()`**  
- Создаёт `NetworkManager` с коллбэком `handle_message`.  
- Запускает приём сообщений.  
- Обрабатывает ввод в формате `<IP> <сообщение>`.
